# docs/05_shared/init.md

## 문서 목적

이 문서는 이번 라운드의 공통 기준 문서다.

이 문서에는 아래 내용을 한 곳에 모아 관리한다.

- 도메인 간 공통 계약
- 아직 확정되지 않은 항목
- 이미 확정된 핵심 결정
- 이번 라운드 완료 기준

이 프로젝트는 토이 프로젝트 수준의 빠른 협업을 전제로 하므로,
공통 문서를 여러 개로 나누지 않고 이 문서 하나로 관리한다.

---

## 현재 프로젝트 상황

현재 프로젝트는 아래 상태에 있다.

- 데이터 / 모델 / 백엔드 / 프론트엔드 각 담당자가 역할별 프롬프트 초안을 작성한 상태다.
- 다음 단계는 각 역할별 문서를 합쳐 공통 계약을 정리하고,
  Codex를 사용해 병렬 코드 생성을 시작하는 것이다.

따라서 현재 가장 중요한 것은
**구현 상세를 늘리는 것보다 도메인 간 입력/출력 계약을 맞추는 것**이다.

---

## 공통 계약

### 1. 모델 관련 공통 계약

- 예측 모델은 반드시 **LSTM 기반**으로 한다.
- 오일 종류와 무관하게 **동일한 하이퍼파라미터 정책**을 사용한다.
- 추론 출력은 미래 예측 시점별 값을 포함해야 한다.
- 시스템은 아래 값을 계산할 수 있어야 한다.
  - 미래 예측 구간 내 최대 예측 유가
  - 미래 예측 구간 내 최소 예측 유가
  - 현재 시점의 구매 판단 근거

---

### 2. recommendation 관련 공통 계약

- recommendation 로직의 최종 판단은 **백엔드**가 수행한다.
- 프론트엔드는 recommendation 결과를 계산하지 않고 **표시**만 수행한다.
- recommendation 응답에는 최소한 아래 정보가 포함되어야 한다.
  - 현재 판단 시점
  - 현재 가격 또는 현재 기준값
  - 최대 예측 유가
  - 최소 예측 유가
  - 추천 결과
  - 추천 이유 요약

---

### 3. 프론트엔드 관련 공통 계약

프론트엔드는 최소한 아래를 표시할 수 있어야 한다.

- 현재 판단 시점
- 예측 차트
- 최대 예측 유가
- 최소 예측 유가
- 구매 추천 결과
- 추천 이유 요약
- 모델/모니터링 상태

프론트엔드는 threshold 계산이나 recommendation 판단 로직을 직접 구현하지 않는다.

---

### 4. 실행 환경 공통 계약

- 데이터 / 모델 / 백엔드는 동일한 Python 가상환경 `ener` 를 사용한다.
- Python 버전은 3.11 기준으로 맞춘다.
- 공용 Python 의존성은 루트 `requirements.txt` 로 관리한다.
- 프론트엔드는 별도의 Node.js 환경을 사용한다.

## 현재 미정 사항

아래 항목은 아직 확정되지 않았거나, 도메인 간 합의가 필요한 항목이다.

### 데이터
- 오일 종류를 구분하는 기준 컬럼
- 날짜 컬럼 및 시계열 정렬 기준
- 결측치 처리 방식
- train / validation / test split 기준

### 모델
- 입력 시퀀스 길이
- 예측 horizon 길이
- 모델 출력 포맷의 구체적인 필드 구조
- MLflow에 기록할 핵심 metric 목록

### 정책
- threshold 기준 방식
  - 절대값 기준인지
  - 상대 비교 기준인지
  - 둘의 조합인지
- recommendation 결과를 2단계로 할지, 다단계로 할지
- 재학습 trigger 기준의 최소 범위

### 백엔드 / 프론트엔드
- monitoring API 범위
- recommendation 응답의 최종 필드명
- 프론트에 표시할 설명 문구 수준

미정 사항은 구현 중 임의로 확정하지 말고,
먼저 팀 합의 후 문서에 반영한다.

---

## 현재 확정된 핵심 결정

아래 항목은 현재 기준으로 확정된 원칙이다.

1. 예측 모델은 **LSTM 기반**으로 간다.
2. 오일 종류와 무관하게 **동일한 하이퍼파라미터 정책**을 사용한다.
3. recommendation 로직의 최종 판단은 **백엔드**에 둔다.
4. 프론트엔드는 recommendation 결과를 **표시**하는 역할에 집중한다.
5. UI는 최소한 아래를 보여야 한다.
   - 현재 시점
   - 최대 예측 유가
   - 최소 예측 유가
   - 추천 결과

## 이번 라운드 완료 기준

이번 라운드는 아래를 만족하면 완료로 본다.

- 데이터 / 모델 / 백엔드 / 프론트엔드 `init.md`를 모두 검토했다.
- 도메인 간 공통 계약 충돌을 정리했다.
- 병렬 구현 전에 필요한 최소 계약이 정리되었다.
- Codex 실행용 통합 프롬프트가 준비되었다.
- 병렬 코드 생성 범위가 정리되었다.
- 병합 순서가 정리되었다.

---

## 병렬 구현 전 최소 체크리스트

아래 항목이 정리되기 전에는 병렬 코드 생성에 들어가지 않는다.

- [ ] 데이터 입력 구조가 정리되었다.
- [ ] 모델 출력 구조가 정리되었다.
- [ ] 예측 horizon이 정리되었다.
- [ ] threshold 정책이 최소 수준으로 정리되었다.
- [ ] recommendation 응답 구조가 정리되었다.
- [ ] 프론트엔드가 필요한 API 필드가 정리되었다.

## 문서 갱신 규칙

아래 상황이 생기면 이 문서를 갱신한다.

- 공통 계약이 변경될 때
- 미정 사항이 새로 생길 때
- 미정 사항이 확정될 때
- recommendation 구조가 바뀔 때
- 프론트 / 백엔드 계약이 바뀔 때
- Codex 실행 전에 공통 기준이 달라질 때
