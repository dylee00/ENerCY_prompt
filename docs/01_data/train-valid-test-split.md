# train-valid-test-split.md

## 문서 목적

이 문서는 모델 학습을 위한 데이터 분리 기준과 슬라이딩 윈도우 생성 정책을 정의한다.

---

## 기본 원칙

- 시계열 데이터의 특성상 **반드시 시간 순서 기반으로 분리**한다.
- Random shuffle을 절대 적용하지 않는다.
- 과거 → 미래 순서를 유지한다.

---

## 데이터 분리 비율

| 구간 | 비율 | 약 데이터 수 | 역할 |
|------|------|------------|------|
| Train | 75% | ~4,500개 | 모델 학습 |
| Validation | 10% | ~600개 | 하이퍼파라미터 튜닝 및 Early Stopping |
| Test | 10% | ~600개 | 최종 성능 평가 |
| Hold-out | 5% | ~300개 | 재학습 트리거 모니터링용 |

---

## 시간 순서 기준 분리

```
2000 ──────────────────────────────────────── 2024
|----------Train(75%)----------|Val(10%)|Test(10%)|HO(5%)|
```

- 각 오일별로 독립적으로 분리한다.
- 오일별 시작일이 다르므로 각 오일 데이터 기준으로 비율을 적용한다.

---

## Hold-out 최소 요건

Hold-out 데이터는 재학습 트리거 모니터링에 사용되므로 아래 최소 조건을 만족해야 한다.

```
최소 필요 데이터 수 = Lookback Window + 예측 구간 끝 시점
                   = 60일 + 23일
                   = 83일 이상
```

- Hold-out 약 300개는 최소 요건(83일)의 3배 이상으로 충분하다.

---

## Lookback Window 및 예측 구간 정의

### Lookback Window

- 정의: 모델 입력으로 사용하는 과거 데이터 기간
- 기준: **60 거래일**
- 이유: 예측 horizon(23일)의 약 2.5배로, 충분한 과거 패턴을 학습하면서 과도한 노이즈를 방지

### 예측 구간 (Prediction Horizon)

- 배송 리드타임: **15일**
- 예측 구간: **Day 16 ~ Day 23 (8일)**
- 이유: 오늘 구매 결정 시 실제 도착 이후 약 1주일(달력 기준)간의 가격 예측

```
오늘(Day 0) 기준
├── Day 1  ~ Day 15 : 배송 기간 (예측 불필요)
└── Day 16 ~ Day 23 : 예측 구간 (8개 값) ✅
```

---

## 슬라이딩 윈도우 생성 방식

- 입력(X): 과거 60 거래일치 feature 데이터
- 출력(Y): Day 16 ~ Day 23의 target 값 (8개)

```
예시:
X = [1일 ~ 60일] → Y = [76일 ~ 83일]
X = [2일 ~ 61일] → Y = [77일 ~ 84일]
X = [3일 ~ 62일] → Y = [78일 ~ 85일]
...
```

---

## Validation 사용 목적

- MLflow 기반 하이퍼파라미터 튜닝 시 성능 기준으로 사용
- LSTM 학습 중 Early Stopping 기준으로 사용
- 과적합 여부 모니터링
